# Example Package Generation and Manifests Quickstart

DIAAD can generate synthetic example file packages for learning command inputs, outputs, configuration, and project structure. These packages are generated by the `examples` command in the CLI or by the dedicated examples download action in the web app.

## Generate Examples

Generate the full example dataset:

```bash
diaad examples
```

Generate a command-specific package:

```bash
diaad examples --for-command "vocab check"
```

Generate one package for a small command set by repeating `--for-command`:

```bash
diaad examples --for-command "cus analyze" --for-command "cus evaluate"
```

In the web app, use Part 3. With no commands selected, the examples button downloads the full example dataset. With commands selected, it downloads examples for the selected functions.

## Package Types

Full example packages are comprehensive atlases:

```text
example_files_full_dataset/
  README.md
  config/
  input/
  expected_outputs/
  example_manifest.json
```

Command-specific packages are smaller toy projects:

```text
example_files_<slug>/
  README.md
  example_config/
  example_input/
  example_output/
  example_logs/
  example_manifest.json
```

The `example_manifest.json` file is the package index. It records commands, command IDs, input/output artifacts, runtime display paths, and package layout.

## What To Inspect

Start with:

- `README.md` for package orientation;
- `example_manifest.json` for machine-readable artifact lists;
- the config directory for runnable settings;
- the input and output folders for concrete file layout examples.

Example data are fully synthetic. They are not human-subjects data, clinical records, or deidentified real transcripts.
