# Example Package Generation and Manifests Usage Guide

Example packages make DIAAD's file contracts tangible before users work with study data. They are especially useful when learning a new command, diagnosing expected folder structure, or teaching the difference between full-project examples and command-specific toy datasets.

## Full Dataset Package

Use the full dataset when you want an atlas of DIAAD examples:

```bash
diaad examples
```

The full package is written inside the timestamped DIAAD output directory as:

```text
example_files_full_dataset/
```

Its main folders are:

```text
config/
input/
expected_outputs/
```

The `expected_outputs/` tree is grouped by module and command. It is intentionally comprehensive, but it does not look exactly like the output folder from one ordinary user workflow. Treat it as a browsable atlas.

## Command-Specific Packages

Use `--for-command` when you want a smaller package that more directly resembles one command's expected use case:

```bash
diaad examples --for-command "transcripts tabularize"
```

Command-specific packages use `example_` prefixes:

```text
example_config/
example_input/
example_output/
example_logs/
```

Those prefixes are intentional. They help users avoid confusing synthetic toy files with real DIAAD project data.

For multiple commands, repeat the flag:

```bash
diaad examples --for-command "transcripts tabularize" --for-command "templates utterances"
```

Do not put a comma-separated command list inside one `--for-command` value. Each flag value should be one canonical DIAAD command.

## Capability Reuse

Command-specific packages are capability-based. If one requested command produces an artifact that another requested command needs, the package can reuse that prior output rather than materializing duplicate inputs.

For example, a combined package for `transcripts tabularize` and `templates utterances` can use the generated transcript table output from the first command as the capability needed by the second command.

`templates subset` is a special case. Its example must currently be generated by itself because that command expects exactly one Excel workbook under `example_input/`.

## Configuration Behavior

There are two configuration layers to keep separate:

- the configuration used by the `diaad examples` run itself;
- the runnable configuration files placed inside the generated example package.

In the CLI, `--config`, `--input-dir`, and `--output-dir` affect where the `diaad examples` run writes its package. The contents of the generated package come from DIAAD's packaged example specs under `src/diaad/examples/assets/spec/`.

Example package configs are not a verbatim copy of `default_config.yaml`. They use demonstration settings such as:

- `reliability_fraction: 0.34`;
- `shuffle_samples: false`;
- `exclude_speakers: [INV]`;
- `num_bins: 2`;
- `num_coders: 2`;
- `stimulus_column: stimulus`;
- synthetic `metadata_fields`;
- `automate_powers: false`;
- `auto_blind: true`;
- a synthetic custom target-vocabulary resource path.

Use Configuration (`docs/manual/02_operation/04_configuration.md`) for DIAAD's packaged defaults.

## Manifest Files

Every generated package includes:

```text
example_manifest.json
```

Full-dataset manifests include:

- `package_kind: full_dataset`;
- `workflow_id: full_example_dataset`;
- `command_id: examples`;
- covered commands and command IDs;
- config and input artifacts;
- expected-output atlas artifacts;
- runtime display paths for normal DIAAD output locations.

Command-specific manifests include:

- `package_kind: command_specific`;
- command metadata;
- command IDs;
- required capabilities;
- suggested invocation;
- `example_config`, `example_input`, `example_output`, and `example_logs` directories;
- config, input, output, and log artifacts.

The manifest is useful for programmatic inspection, tests, and documentation tooling. Most users should read the README first, then use the manifest as a precise artifact index.

## Web App Downloads

The web app has a dedicated examples download action in Part 3. It is not an ordinary dispatch command checkbox.

If no commands are selected, the web app downloads the full example dataset. If commands are selected, it downloads command-specific examples for those selected commands. ZIP names use stable slugs such as:

```text
diaad_example_files_cus_analyze_cus_evaluate_YYMMDD_HHMM.zip
```
