# Configurable Sample and Utterance Identifiers Research Context

Identifiers are methodological infrastructure. They determine whether coding decisions, transcript rows, reliability judgments, metadata, and analysis summaries can be traced back to the same analytic units.

## Stable Coordinates

A discourse workflow often produces many files from the same samples. Stable identifiers let those files remain interoperable without relying on fragile row order or file-name memory.

The sample identifier anchors the transcript or session. The utterance identifier anchors a row within that sample. Together, they support an audit trail from transcript text to coding and analysis.

## Why Defaults Are Usually Best

Defaults reduce cognitive load and make the manual, generated examples, tests, and command pages easier to apply. Custom identifiers are valuable when integrating external datasets, but they add a consistency requirement: every manually edited workbook, codebook, and analysis input must use the same names.

## Revision Risk

Changing identifiers after coding has started is more serious than changing a display label. It can sever the connection between a coder's decision and the transcript row it described. In some cases, the safest response is to regenerate coding files and recode affected rows.
