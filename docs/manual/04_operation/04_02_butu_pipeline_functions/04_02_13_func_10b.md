# Run Target Vocabulary Coverage Analysis

*Pipeline context:* Stage 10.  
*General purpose:* Compute target vocabulary coverage using built-in CoreLex-style resources and optional norm percentiles; optionally incorporate speaking rate (from *4a Samples*).

*Inputs*

1. main path: CU summaries (from *10a*) & transcript tables with speaking times (from *4a*).  
2. minimal path: analyze built-in target vocabulary coverage directly on `cha` inputs, or run *4a* and optionally complete the speaking_time column.

*Outputs*

- Target vocabulary coverage workbook with a summary sheet and one long-format detail sheet.

*CLI*

- Succinct: `diaad 10b`
- Expanded: `diaad "corelex analyze"`

*Streamlit*

- “*10b. Run CoreLex analysis*”

*Manual step associated*

- None; ensure prerequisite summaries exist.
