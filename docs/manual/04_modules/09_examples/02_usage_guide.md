# Examples Module Usage Guide

The Examples module has two user-facing package modes: full-dataset examples and command-specific examples. It also has a maintainer-facing documentation regeneration mode.

## Full Dataset Examples

`diaad examples` creates a full synthetic example dataset. In a CLI run, it is written inside the current timestamped DIAAD output folder as:

```text
example_files_full_dataset/
```

The full package is best when you want to browse DIAAD's overall file organization and see how commands relate across modules.

The full package includes input fixtures and expected outputs for many commands. It is a teaching fixture rather than a template that every user project must copy exactly. Its `expected_outputs/` tree is a full-dataset atlas grouped by module and command, not the exact output tree of a normal single DIAAD workflow.

## Command-Specific Examples

Use `--for-command` when you want a smaller package for one command or a small set of commands:

```bash
diaad examples --for-command "cus analyze" --config config
```

Repeat `--for-command` for a command set:

```bash
diaad examples --for-command "cus analyze" --for-command "cus evaluate" --config config
```

Command-specific packages are meant to show the minimal configuration, input, output, and logs needed to understand that command. They use:

```text
example_config/
example_input/
example_output/
example_logs/
```

These names intentionally use `example_` prefixes so users do not confuse synthetic toy data with real project data. The output folder mirrors normal runtime output structure more directly than the full-dataset atlas does.

`templates subset` examples must currently be generated by themselves because that command expects exactly one Excel workbook under `example_input/`.

## Web App Downloads

The web app does not expose `examples` as an ordinary selectable dispatch command. In Part 3: Commands, it provides a dedicated example-file download action.

If no commands are selected, the button downloads the full example dataset. If commands are selected, the button downloads command-specific examples for the selected functions.

The web app ZIP filename uses a stable example slug, such as:

```text
diaad_example_files_cus_analyze_cus_evaluate_YYMMDD_HHMM.zip
```

## Example Configs

Example packages include their own runnable configuration files. Command-specific packages use:

```text
example_config/project.yaml
example_config/advanced.yaml
```

Those configs are generated from DIAAD's packaged synthetic example specs. They are not simply a copy of the installed default configuration. For example, the specs use demonstration values for reliability fraction, bins, coder count, metadata fields, blinding, and target-vocabulary resources.

The user's `--config` or webapp configuration controls where the `diaad examples` run writes its package, but the contents of the package are generated from the example specs.

For default DIAAD configuration values, see Configuration (`docs/manual/02_operation/04_configuration.md`).

## Example I/O Documentation

Generated Example I/O pages under `src/diaad/examples/assets/rendered_docs/example_io/` are documentation views. They show representative command inputs, outputs, snippets, and preview tables.

The generated docs and downloadable toy packages are aligned by command identity, but they do not have to match exactly. The docs are optimized for reading inside the manual; the packages are optimized for hands-on exploration.

## Force And Regeneration

Use `--force` when an existing example package should be replaced. Use it cautiously if you have copied or edited example outputs for local notes.

`--render-docs` is for documentation maintenance. It regenerates packaged Example I/O markdown under `src/diaad/examples/assets/rendered_docs/example_io/`; it is not needed for ordinary use.

## Do Not Edit Generated Views By Hand

Generated Example I/O pages are maintained through specs, generator code, renderer code, and tests. Edit those sources and regenerate rather than copying generated tables into authored manual pages.
