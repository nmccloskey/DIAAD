# Examples Research Context

The Examples module is not a research measurement module. Its research value is indirect: it supports reproducibility, pedagogy, and transparent documentation by giving users synthetic, runnable files with known structure.

## Why Synthetic Examples Matter

Discourse-analysis software can be difficult to learn from prose alone because file layout, workbook sheets, command sequencing, and output naming all matter. Synthetic examples let users inspect those details without handling sensitive participant or clinical data.

They also let maintainers keep documentation examples reproducible. Generated Example I/O pages can be rebuilt from packaged specs and generated artifacts rather than manually copied into the authored manual.

## Limits

Synthetic examples are deliberately small. They demonstrate file contracts and command behavior, not substantive research results. Users should not infer population-level, clinical, or psychometric claims from example outputs.
