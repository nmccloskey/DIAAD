# `templates combine` Usage Guide

Use `diaad templates combine` when several workbook files should become one workbook without changing the user-entered data columns.

## Input Directory

This command accepts any `.xlsx` filename. Unlike `templates subset`, it expects multiple input workbooks when there are multiple files to combine.

A common layout is:

```text
diaad_data/input/template_coding/
  coder_a/
    coding.xlsx
  coder_b/
    coding.xlsx
```

with:

```yaml
input_dir: diaad_data/input/template_coding
```

Duplicate filenames are allowed because `source_file` records each file by its path relative to `input_dir`, such as `coder_a/coding.xlsx` and `coder_b/coding.xlsx`.

## Schema Requirements

DIAAD reads every `.xlsx` workbook under `input_dir`. All workbooks must contain the same sheet names. For each sheet, all workbooks must contain the same column names.

Column order may vary across files. DIAAD writes columns in the order used by the first discovered workbook for that sheet.

Empty sheets are allowed when the sheet has the expected headers. They still appear in the `metadata` sheet with `num_rows` set to `0`.

## Reserved Names

Input workbooks may not contain a sheet named `metadata`, because DIAAD uses that sheet name for combination provenance.

Input sheets may not already contain:

```text
combined_id
source_file
```

Those columns are generated by the combine command.

## Output Workbook

The command writes:

```text
coding_templates/combined.xlsx
```

For each input sheet, DIAAD stacks the corresponding dataframes in discovery order. Each combined sheet starts with `combined_id` and `source_file`, followed by the original input columns.

The `metadata` sheet records:

| Column | Meaning |
|---|---|
| `source_file` | Input workbook path relative to `input_dir`. |
| `order` | Workbook order used during stacking. |
| `sheet` | Input sheet name. |
| `num_rows` | Number of rows read from that workbook and sheet. |

## Common Problems

If DIAAD reports a sheet mismatch, compare the workbook tabs across files.

If DIAAD reports a column mismatch, compare the headers within the named sheet. A missing blank header, renamed coding column, or accidental extra notes column can cause the command to stop.

If a workbook is open in Excel, DIAAD ignores the temporary `~$...xlsx` lock file, but the real workbook still needs to be readable.
