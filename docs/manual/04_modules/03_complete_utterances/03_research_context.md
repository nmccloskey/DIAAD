# Complete Utterances Research Context

Complete Utterance coding is a manual discourse-analysis paradigm. DIAAD supports the workflow by organizing transcript-derived utterances into coding files, selecting reliability material, evaluating agreement, and aggregating completed coding.

The software does not replace a coding protocol. A project still needs explicit rules for deciding what counts as a Complete Utterance and how ambiguous cases should be handled.

## Why DIAAD Helps

Manual coding workflows are vulnerable to small reproducibility failures: inconsistent file versions, missing identifiers, unclear reliability subsets, and analysis tables that cannot be traced back to the coded utterances. DIAAD reduces those risks by keeping CU workbooks connected to transcript-table identifiers and by using consistent reliability and analysis outputs.

The module is designed for human review and coder judgment. It can organize rows, apply coder assignment, support blinding, and summarize completed values, but the coding decision remains a research-protocol decision.

## Reliability And Reselection

CU reliability workflows compare primary and reliability coding. Reselection exists as a fallback when additional reliability material is needed after a prior round. Later command pages should explain exact workbook requirements and evaluation metrics.

## Rates

CU rates use speaking-time values to normalize count-like CU summaries per minute. The numerator depends on the completed coding and analysis output; the denominator comes from a speaking-time workbook.
