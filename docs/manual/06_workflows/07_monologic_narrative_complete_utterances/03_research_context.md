# Monologic Narrative Complete Utterances Research Context

Complete Utterance coding is a human judgment workflow. DIAAD makes the workflow more organized and auditable, but it does not supply the coding protocol.

## Protocol Centrality

CU coding depends on decisions about grammaticality, relevance, completeness, and ambiguous cases. Those decisions should be specified in a project protocol before coding begins. If a project has dialect-sensitive criteria, the coding workbook can include paradigm-specific columns, but the interpretation of those columns belongs to the protocol.

## Why Use Transcript Tables

Transcript tables keep CU coding attached to stable sample and utterance identifiers. This makes it easier to:

- distribute coder-facing rows;
- select reliability material;
- compare primary and reliability coding;
- trace analysis summaries back to utterances;
- revise transcript rows carefully when coding reveals transcript errors.

## Reliability

Reliability is a first psychometric hurdle for manual coding workflows. Low agreement can reflect coder error, ambiguous rules, transcript problems, or an insufficiently operationalized construct. The reliability output should therefore be used diagnostically, not only as a pass/fail summary.

## Rates

CU rates normalize count-like summaries by speaking time. They help compare samples of different duration, but they do not replace raw counts or protocol-level interpretation.

## Draft Review Notes

Before publication, review CU protocol wording, dialect-column examples, and any reliability-threshold language against the project's intended methods language.
